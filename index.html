COBug

COBug - A Bug Localization Pipeline for COBOL Software Systems


Tool Description

COBOL continues to power critical systems in financial, governmental, and enterprise applications. However, bug localization for COBOL systems has not been addressed effectively in research. This tool is the first of its kind to automatically identify buggy COBOL files given a natural language bug report, offering a significant improvement to the maintenance of COBOL systems.

Installation

Requirements

  • Install Python ≥ 3.9 (Recommended: 3.11) and pip ≥ 21.0
  • Download and install Python from the official site.
  • Ensure you check "Add Python to PATH" during installation.
  • Verify the installation by running python --version and pip --version in your terminal or command prompt.
  • Obtain the Gemini 2.0 Flash API key for bug report generation:
    • Go to Google AI Studio - API Key Management.
    • Sign in with your Google Account.
    • Click Create API Key and select Gemini 2.0 as the model.
    • Copy the generated API key and keep it safe.
    • To verify your API access, run the following Python snippet: 
      import google.generativeai as genai
genai.configure(api_key="YOUR_API_KEY")
model = genai.GenerativeModel("models/gemini-2.0-flash")

From Github

STEP 1: Clone or download this GitHub repository:

git clone https://github.com/rishalab/COBug.git

STEP 2: Navigate into the COBug directory:

cd COBug

STEP 3: Install the requirements:

python -m pip install -r requirements.txt

STEP 4: Run the tool:

python main.py

After running, the tool will ask for the following inputs:

STEP 5: Provide the path to the COBOL repository you want to analyze, e.g.,

path/to/cobol_repo

STEP 6: Enter your API keys for report generation when prompted.

STEP 7: To test the tool's prediction task, the user should provide the following inputs when prompted:

  • Bug summary: Enter the summary of one bug ID.
  • Bug description: Enter the detailed description of the bug.
  • Ground truth (optional):Provide the actual cbl path for comparison purposes.

After submitting these inputs, the tool will process the information and display the results, showing the top 5 predicted COBOL files that are most relevant to the bug report.

This allows users to quickly identify which COBOL files are likely related to the bug, aiding in faster debugging and analysis.

Usage

COBug helps developers working with COBOL code quickly find where bugs are likely hiding. Just give it your codebase, a brief summary and description of the bug, and an API key—it will analyze everything and show you the top files most likely causing the issue.
